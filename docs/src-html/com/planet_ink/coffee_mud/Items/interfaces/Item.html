035/**
036 * The interface for all common items, and as a base for RawMaterial, armor, weapons, etc.
037 *
038 * @author Bo Zimmerman
039 *
040 */
041public interface Item extends Rider, DBIdentifiable, Wearable, PhysicalAgent, Readable
042{
043        /**
044         * Where the item is located.  Either null for
045         * plain site (or contained on person), or will
046         * point to the container object
047         * @return The item in which  it is contained, or null.
048         */
049        public Container container();
050        /**
051         * Change  the container where the item is located.  Either null for
052         * plain site (or contained on person), or will
053         * point to the Container item.
054         * @see Container
055         * @param newLocation Container item in which this item is contained.
056         */
057        public void setContainer(Container newLocation);
058        /**
059         * If an item is in a container, which is in a container, etc, this will
060         * return the "highest" or ultimate container in which this item is located.
061         * If an item is in a container which is in plain view, it will simply
062         * return container().  If the item is not in a container, it will return itself.
063         * @param stopAtC if contained in this object, it will return this object.. default null.
064         * @see Container
065         * @return the highest level container in which  this item is found, or itself
066         */
067        public Item ultimateContainer(Physical stopAtC);
068        /**
069         * This method basically calls setContainer(null), and then removes this item
070         * from its owner().  It effectively removes the item from the map.  This is
071         * generally assumed to be a temporary condition.  To really destroy the item
072         * permanently, the destroy() method is used.  The unWear() method is also called.
073         * @see Container
074         */
075        public void removeFromOwnerContainer();
076
077        /**
078         * How many items this Item object represents.  When an item is Packaged, this
079         * method will return a number greater than 1, otherwise it always returns 1.
080         * @return the number of items represented by this object.
081         */
082        public int numberOfItems();
083
084        /**
085         * This method returns the calculated and expanded description of the properties
086         * of the item as would be discovered through the Identify spell.  It starts with
087         * its rawSecretIdentity() and adds to it any strings which the Ability objects
088         * contained in the Items effects list would generate.  An empty string means
089         * the item has no secret properties per se.
090         * @return a displayable string describing the secret properties of the item.
091         */
092        public String secretIdentity();
093        /**
094         * This method returns those secret properties of the item which are entered directly
095         * by the builder when the item is designed.  It is the string saved to the database,
096         * and is used by the secretIdentity() method to construct a full secret description
097         * of the Item.
098         * @return the string entered by the builder as the item secret properties or name.
099         */
100        public String rawSecretIdentity();
101        /**
102         * This method is used to change the string returned by rawSecretIdentity.  This string
103         * is saved to the database as the items secret properties desctiption.  The secretIdentity
104         * method uses this string to construct its full description.
105         * @param newIdentity the secret properties of this item.  Empty string means it has none.
106         */
107        public void setSecretIdentity(String newIdentity);
108
109        /**
110         * Whether the usesRemaining() number above is used to determine the percentage health of
111         * the item.  If this method returns false, then health or condition is irrelevant to this
112         * Item. If true is returned, then usesRemaining is a number from 100 to 0,  where 100 means
113         * perfect condition, and 0 means imminent disintegration.
114         * @return whether this item has a valid condition
115         */
116        public boolean subjectToWearAndTear();
117        /**
118         * Uses remaining is a general use numeric value whose meaning differs for different Item
119         * types.  For instance, Wands use it to represent charges, Weapons and Armor use it to
120         * represent Condition, Ammunition uses it to represent quantity, etc.
121         * @return the general numeric value of this field.
122         */
123        public int usesRemaining();
124        /**
125         * Sets the uses remaining field, which is a general numeric value whose meaning differs
126         * for different Item types.  See usesRemaining() method above for more information.
127         * @param newUses a new  general numeric value for this field.
128         */
129        public void setUsesRemaining(int newUses);
130
131        /**
132         * If this Item is current Ticking due to its having Behaviors or other properties which
133         * might grant it the ability to Tick, this method will cause that ticking to cease and
134         * desist.  This means that it will lose its periodic thread calls to its tick() method.
135         * This method also makes the item appear destroyed without actually being destroyed.
136         * The whole idea is to make the item 'dormant' for storage.
137         */
138        public void stopTicking();
139        /**
140         * The default value of the item, represented in the base CoffeeMud currency.  This
141         * method starts with baseGoldValue, which is a user-entered value, and adjusts
142         * according to magical enhancements and the condition of the  item.
143         * @return the adjusted value of the item in the base currency.
144         */
145        public int value();
146        /**
147         * The user/builder-entered value of the item, represented in base CoffeeMud currency.
148         * It is used as a basis for the value returned by the value() method.
149         * @return the raw user-entered value of item.
150         */
151        public int baseGoldValue();
152        /**
153         * Changes the base value of the item, represented in base CoffeeMud currency.
154         * The value is saved to the database, and is used by the value() method as a basis.
155         * @param newValue the new raw value of the item
156         */
157        public void setBaseValue(int newValue);
158
159        /**
160         * The resource code representing the material out of which this item is principally made.
161         * The resource codes are composed of an integer where the highest order bits represent
162         * the basic material type, and the lower order bits represent the specific material type.
163         * These codes are defined in RawMaterial.
164         * @see RawMaterial
165         * @return the RawMaterial code describing what this item is made of.
166         */
167        public int material();
168        /**
169         * Sets the resource code representing the material out of which this item is principally made.
170         * The resource codes are composed of an integer where the highest order bits represent
171         * the basic material type, and the lower order bits represent the specific material type.
172         * These codes are defined in RawMaterial interface.
173         * @param newValue the resource code
174         * @see RawMaterial
175         */
176        public void setMaterial(int newValue);
177
178        /**
179         * For a normal item, this method returns the same as phyStats().weight().  For
180         * a Container, it returns the weight of the container plus the recursive weight
181         * of all items in the container.
182         * @see Container
183         * @return the total weight of the item and any possible contents.
184         */
185        public int recursiveWeight();
186        /**
187         * The Room or MOB representing where this item is located.  Containers are handled
188         * by another pointer, container(), so those two methods be used together to determine
189         * where a given item is.
190         * @see com.planet_ink.coffee_mud.MOBS.interfaces.MOB
191         * @see com.planet_ink.coffee_mud.Items.interfaces.Item
192         * @return the mob or room where the item is located
193         */
194        public ItemPossessor owner();
195        /**
196         * Sets the Room or MOB representing where this item is located.  Containers are handled
197         * by another pointer, container(), so those two methods be used together to determine
198         * where a given item is.  This method is called by the addItem method on mobs
199         * and the addItem interface on Rooms.  Alone, this method is insufficient to properly
200         * determine an items location, so one of the two above should be called instead.
201         * @see com.planet_ink.coffee_mud.core.interfaces.ItemPossessor#addItem(Item)
202         * @param E the mob or room where the item is located
203         */
204        public void setOwner(ItemPossessor E);
205}
