036/**
037 * The ServerUser interface prescribes behaviours which
038 * must be defined by any user connection used with
039 * the Imaginary JavaMud Server.  Specifically, it
040 * requires that the user connection be able to handle
041 * user input somehow.
042 * Created: 27 September 1996
043 * Last modified: 27 Septembet 1996
044 * @author George Reese (borg@imaginary.com)
045 * @version 1.0
046 */
047public interface ServerUser extends ServerObject {
048        /**
049         * This method is triggered by the server when the
050         * user first connects.
051         */
052        public abstract void connect();
053
054        /**
055         * The server calls this method every server cycle.  The
056         * mudlib implementation is expected to be queueing up user
057         * input as it gets it (as opposed to processing it immediately
058         * as it comes across the net) for synchronicity's sake.
059         * The mudlib implementation therefore should use this method
060         * to pull a command off the queue and process it.
061         */
062        public abstract void processInput();
063
064        /**
065         * The server calls this method just after creating an instance
066         * of the mudlib user connection object that implements this
067         * interface.  Normally, the socket would be passed as an
068         * argument to the constructor.  Because the Server class does
069         * not know the name of the user connection implementation class
070         * at compile time, it has to use the Class.forName.newInstance()
071         * construct, which means the default constructor must be used.
072         * This method thus allows the server to pass the mudlib
073         * implementation the socket to use for communication with the
074         * client.
075         * @exception java.io.IOException thrown if a problem creating I/O streams occurs
076         * @param s the socket connected to the user's machine
077         * @see java.lang.Class#forName
078         * @see java.lang.Class#newInstance
079         */
080        public abstract void setSocket(java.net.Socket s) throws java.io.IOException;
081}
